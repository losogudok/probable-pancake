# Научный клубок — Scientific Knowledge Graph (GraphRAG)

**Платформа гибридного поиска и генерации ответов с графом знаний для научно-исследовательских работ в области горной промышленности и цветной металлургии.**

## Установка и запуск

### Требования

- Python 3.10+
- CUDA (опционально — для GPU-ускорения LLM)

### Установка

```bash
git clone <repo-url> scientific_kg
cd scientific_kg
pip install -r requirements.txt
```

### Сборка индекса

```bash
export PYTHONPATH=$PWD:$PYTHONPATH
python3 -m app.build
```

Параметры:
- `--max-docs N` — ограничить количество документов (для теста)
- `--use-llm` — включить LLM-извлечение сущностей
- `--llm-sample-rate N` — частота LLM-обработки (по умолч. каждый 10-й чанк)

### Запуск сервера

```bash
cd frontend
npm ci
npm run build
cd ..
uvicorn app.api.server:app --host 0.0.0.0 --port 8080
```

### Веб-интерфейс

Откройте `http://localhost:8080` в браузере.

Для frontend-разработки без backend используется типизированный mock API:

```bash
cd frontend
cp .env.example .env
npm run dev
```

Откройте `http://localhost:5173`. Для переключения на HTTP-адаптер задайте
`VITE_API_MODE=http` и `VITE_API_BASE_URL=http://localhost:8080`.

### Пример запроса RAG

```bash
curl -X POST http://localhost:8084/api/ask \
  -H "Content-Type: application/json" \
  -d '{"query":"Какие методы обогащения никель-содержащих руд существуют?"}'
```

Ответ включает:
- `answer` — структурированный ответ на русском
- `sources` — процитированные источники с оценками
- `numerics` — числовые параметры из ответа
- `contradictions` — обнаруженные противоречия
- `experts` — упомянутые эксперты
- `gaps` — пробелы знаний

## Ключевые технические решения

### 1. Двухуровневое извлечение сущностей
Эвристическое (ключевые слова + regex) работает всегда и обеспечивает базовое покрытие. LLM-извлечение запускается на каждом N-ом чанке для повышения качества связей.

### 2. Гибридный поиск
BM25 обеспечивает точное совпадение термов (хим. формулы, названия параметров). Векторный поиск (rubert-tiny2, 312d) — семантическое совпадение через RU/EN синонимы.

### 3. NetworkX вместо Neo4j
Для демо-версии граф в памяти на NetworkX достаточен (100ms на запросы для 10K узлов). JSON-персистентность без внешних зависимостей.

### 4. Инкрементальная сборка
При повторном запуске обрабатываются только новые/изменённые файлы (по ID документов), эмбеддинги пересчитываются только для новых чанков.

### 5. Двуязычная поддержка
Все ключевые слова, синонимы и онтология поддерживают RU/EN. Язык документа определяется автоматически.

## Известные ограничения

- **Качество текста из PDF**: некоторые PDF (особенно старые) используют CID-шрифты, текст извлекается с искажением кириллицы
- **Противоречия**: merge-логика `add_relation` перезаписывает значения параметров, поэтому обнаружение противоречий не работает на полную мощность (требует доработки)
- **Авторы**: извлечение авторов из PDF-метаданных ограничено
- **LLM-зависимость**: при недоступности LLM-эндпоинта RAG отвечает только контекстом без генерации
